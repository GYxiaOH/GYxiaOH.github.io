OCR系列(2)-R2CNN、CPTN、FEN

发表于 2019-05-20 | 分类于 深度（机器）学习 | 阅读次数:

最近看的论文越多发现很多论文就没讲清楚最核心的方法到底具体怎么做。。。。。
　
首先是继续跟着Scene Text Detection and Recognition: The Deep Learning Era中Region based的思路，将R2CNN和FEN看完，之后补充了下经典的CPTN算法。

R2CNN

　　　R2CNN是RCNN之后发布的一篇论文，和RCNN思路差不多，也是预测倾斜的框，不过它倾斜的框是用(X1,Y1,X2,Y2,H)表示的，原文是说用角度表示的话90度和-90度的图形差不多可是计算会差的比较多，因此换成用上方的两个点加上高来表示倾斜的矩形。
　　　除此之外，为了适应文字proposal的特点（一般是长宽比较大）,因此在roi部分去了三种不同大小的ROI作为最后的特征做refine。
　　　同时对水平框和倾斜框做回归，这种multi task的做法能使最后的结果更好

FEN

Feature enhancement network: A refined scene text detector
　　这篇就基本是在整个framework上进行了修改，基于的框架是R-FCN主要修改就是三个：

  1. 提出了Feature Enhancement Network，有点像特征金字塔，理由也是最后一层的特征太高级了，对细节太不清楚，因此作者每隔几层抽出来特征进行卷积BNdeconv然后把特征融合在一起作为最后的特征

  2. 与R2CNN一样，在ROI POOLING上选择了不同尺度的大小，不过最后是通过一个所谓自适应的权重来选择最后的ROI POOLING而不是融合，自适应的权重是通过最后每个ROI POOLING的score做一个类似于softmax的操作得到的。（因为R-FCN最后会得到一个C+1的向量表示每个类的score，用这个得分作为权重）

  3. 提出了正样本增强的策略解决样本不平衡的问题。由于一张图片可能文本区域就1两个，所以最后网络输出的正样本可能很少，为了解决这个问题，作者吧RPN最后输出的正样本进行一个scale【0.7,1.3】，这样得到的proposal也算正样本，感觉上像是做了数据增强了一样吧。作者是对排名前50的proposal做了这个增强

参考资料
Feature Enhancement Network: A Refined Scene Text Detector

CTPN

CTPN是早期最为常用效果也不错的一个TEXT DETECTION方案，但是其弱点也很明显，只能检测水平text proposal，因此现在没怎么用了，它由以下几个特点：

  1. 不预测整个文本区域，利用微分的思想，预测固定宽度的小矩形，最后通过合并小矩形得到最后的文本框，因此在RPN时，只回归两个值，真实的y和其高度
  2. 最后输出的三个分支，分别是分类，y和高度的回归，以及side-refinement即水平的偏移，我也不知道为啥它不在一个回归分支里回归
  3. 最后的特征除了视觉特征之外，还有将视觉特征输入双向LSTM后得到的序列特征。
  4. 利用文本构造的方法合并最后的矩形得到最后的文本框。
